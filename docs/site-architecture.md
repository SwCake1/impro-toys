# Архитектура сайта Impro Toys

## Назначение

Сайт содержит простые инструменты для импровизации. Каждый инструмент доступен
на отдельной странице.

Планируемые инструменты:

- звонок;
- генератор эмоций;
- генератор значимых слов;
- генератор локаций;
- генератор отношений между персонажами.

Главная страница служит каталогом и содержит ссылки на все инструменты.

## Технологии

Сайт состоит из обычных HTML-, CSS- и JavaScript-файлов без фреймворков,
сборщика и сторонних зависимостей. Такая структура напрямую поддерживается
GitHub Pages и подходит для небольших независимых инструментов.

## Организация страниц

Каждый инструмент размещается в собственной папке внутри `tools/`. Благодаря
файлу `index.html` страницы получают короткие адреса без имени HTML-файла:

```text
/tools/bell/
/tools/emotion/
/tools/word/
/tools/location/
/tools/relationship/
```

Предлагаемая структура проекта:

```text
impro-toys/
├── index.html
├── tools/
│   ├── bell/
│   │   ├── index.html
│   │   └── app.js
│   ├── emotion/
│   │   ├── index.html
│   │   ├── app.js
│   │   └── emotions.json
│   ├── word/
│   │   ├── index.html
│   │   ├── app.js
│   │   └── words.json
│   ├── location/
│   │   ├── index.html
│   │   ├── app.js
│   │   └── locations.json
│   └── relationship/
│       ├── index.html
│       ├── app.js
│       └── relationships.json
├── shared/
│   ├── styles.css
│   ├── navigation.js
│   └── assets/
│       ├── icons/
│       └── sounds/
├── docs/
│   └── site-architecture.md
└── README.md
```

Папки и файлы создаются по мере появления соответствующих инструментов, а не
заранее.

## Каталог инструментов

Корневой `index.html` содержит только карточки уже реализованных инструментов.
Каждая карточка — одна нативная ссылка, занимающая всю видимую поверхность. В
ней обязательно есть короткое название и собственный графический идентификатор
из `shared/assets/`. Обработчик перехода на JavaScript не добавляется.

Сетка каталога использует две колонки на телефонах. На более широких экранах
CSS Grid автоматически добавляет колонки через `auto-fill`, а центрированный
контейнер ограничивает максимальную ширину. Ссылки на инструменты и общие
ресурсы остаются относительными, чтобы каталог работал при локальном открытии и
на GitHub Pages.

## Общее оформление и навигация

`shared/styles.css` содержит оформление каталога и небольшой переиспользуемый
примитив `.tool-navigation`. Внутреннее оформление инструментов остаётся
самостоятельным.

Каждая страница `tools/<name>/index.html` напрямую включает подписанную ссылку
вида:

```html
<a class="tool-navigation" href="../../index.html">Все инструменты</a>
```

Ссылка ведёт в каталог независимо от истории браузера. Инструмент может
настроить её цвета через CSS-переменные `--tool-nav-foreground`,
`--tool-nav-background`, `--tool-nav-border` и `--tool-nav-focus`, но сохраняет
подпись, прямой относительный URL, заметный фокус и доступное расположение.
Небольшое дублирование этой HTML-разметки предпочтительнее JavaScript-вставки
или добавления сборщика.

При добавлении инструмента нужно одновременно:

1. создать его отдельную страницу в `tools/<name>/`;
2. добавить в корневой каталог полноразмерную ссылку-карточку;
3. добавить на страницу инструмента подписанную ссылку `.tool-navigation`;
4. проверить обе страницы на телефоне и настольном экране с клавиатурой,
   указателем и касанием.

## Данные и логика

Логика и данные конкретного инструмента находятся рядом с его страницей.
Логика хранится в `app.js`, а списки значений — в JSON-файле в папке этого же
инструмента. Например, страница эмоций загружает `tools/emotion/emotions.json` и
случайным образом выбирает одно из значений.

Так списки можно дополнять независимо от программной логики.

## Публикация

Целевой вариант — персональный сайт GitHub Pages из репозитория
`SwCake1/SwCake1.github.io`. Главная страница будет доступна по адресу:

```text
https://swcake1.github.io/
```

Страницы инструментов будут доступны, например, по адресам:

```text
https://swcake1.github.io/tools/emotion/
https://swcake1.github.io/tools/location/
```

Внутренние ссылки и пути к ресурсам должны оставаться относительными, когда это
возможно. Это позволяет открывать страницы локально и упрощает перенос сайта.

## Принятые ограничения

- один инструмент — одна отдельная страница;
- общий стиль и навигация для всех инструментов;
- без frontend-фреймворка и этапа сборки;
- без лишней инфраструктуры до появления реальной необходимости;
- новые папки и данные добавляются постепенно.
