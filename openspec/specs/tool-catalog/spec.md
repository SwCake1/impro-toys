## Requirements

### Requirement: Доступные инструменты представлены запускаемыми карточками
Каталог SHALL представлять каждое доступное мини-приложение в виде карточки, вся видимая область которой служит ссылкой на инструмент. Каждая карточка MUST содержать название инструмента и лаконичный визуальный идентификатор.

#### Scenario: Пользователь открывает доступный инструмент
- **WHEN** пользователь активирует любую область карточки доступного инструмента
- **THEN** браузер переходит на отдельную страницу этого инструмента

#### Scenario: Каталог содержит только реализованные инструменты
- **WHEN** отображается каталог
- **THEN** в нём присутствуют запускаемые карточки только тех инструментов, страницы которых уже доступны

### Requirement: Каталог использует адаптивную многоколоночную сетку
Каталог SHALL отображать ровно две колонки карточек при ширине экрана телефона и SHALL динамически добавлять колонки при появлении свободного горизонтального пространства. Каталог MUST иметь максимальную ширину и располагаться по центру, чтобы карточки не растягивались без ограничений.

#### Scenario: Каталог открыт на телефоне
- **WHEN** ширина области просмотра соответствует телефону
- **THEN** в каждом полном ряду каталога отображаются две карточки

#### Scenario: Каталог открыт на широком экране
- **WHEN** в области просмотра комфортно помещается больше двух карточек заданной минимальной ширины
- **THEN** каталог динамически добавляет колонки без набора жёстких раскладок для конкретных устройств

### Requirement: Каталог визуально чистый и интерактивный
Каталог SHALL использовать простое оформление с лёгким театральным настроением, ясной иерархией и без декоративного шума. Карточки и вспомогательные элементы SHALL заметно реагировать анимацией на действия пользователя, сохраняя читаемость и понятность навигации.

#### Scenario: Пользователь взаимодействует с карточкой
- **WHEN** пользователь наводит указатель, переводит фокус, нажимает или касается карточки доступным на устройстве способом
- **THEN** карточка отвечает анимированной визуальной реакцией, не скрывая подпись и не препятствуя переходу

#### Scenario: Каталог находится в покое
- **WHEN** пользователь не взаимодействует со страницей
- **THEN** интерфейс остаётся читаемым и упорядоченным, а понимание и навигация не зависят от анимации

### Requirement: Каталог работает с указателем, касанием и клавиатурой
Каталог SHALL поддерживать настольные и мобильные браузеры. Каждая карточка инструмента MUST работать с указателем, касанием и клавиатурой благодаря нативной семантике ссылки.

#### Scenario: Пользователь перемещается по каталогу с клавиатуры
- **WHEN** фокус клавиатуры достигает карточки инструмента
- **THEN** карточка получает видимое состояние фокуса и активируется стандартным для ссылки способом

#### Scenario: Пользователь открывает каталог на сенсорном устройстве
- **WHEN** пользователь касается карточки
- **THEN** карточка реагирует и открывает инструмент без зависимости от состояния наведения

### Requirement: Внутренние ссылки каталога переносимы
Каталог SHALL использовать относительные внутренние URL, совместимые с локальным просмотром и размещением на GitHub Pages.

#### Scenario: Сайт размещён в настроенном корне
- **WHEN** пользователь активирует ссылку инструмента в каталоге
- **THEN** переход выполняется внутри этого же сайта без жёстко заданного production-домена
