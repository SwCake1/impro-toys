## Requirements
### Requirement: Генератор предоставляет большую подборку реалистичных отношений
Генератор SHALL содержать не менее 120 содержательно различных отношений между
двумя персонажами. Каждое отношение MUST быть возможным в обычной жизни и MUST
описывать только социальную связь или роли персонажей без конфликта, тайны, цели,
эмоции, действия, обстоятельства сцены, фантастического или абсурдного элемента.

#### Scenario: Подборка проходит содержательную проверку
- **WHEN** проверяется полный набор отношений
- **THEN** он содержит не менее 120 реалистичных вариантов, каждый из которых оставляет развитие сцены игрокам

#### Scenario: В подборке отсутствуют смысловые дубли
- **WHEN** сравниваются формулировки и смысл всех отношений
- **THEN** ни один вариант не повторяет другое отношение только иными словами

### Requirement: Пользователь получает отношения одним основным действием
Генератор SHALL показывать новые отношения между двумя персонажами после
активации основной кнопки «Новые отношения». До первого запуска интерфейс MUST
ясно приглашать пользователя выполнить это действие.

#### Scenario: Пользователь запускает генератор впервые
- **WHEN** пользователь активирует кнопку «Новые отношения» до показа первого результата
- **THEN** интерфейс показывает одну случайно выбранную пару отношений

#### Scenario: Пользователь запрашивает следующий вариант
- **WHEN** пользователь активирует кнопку «Новые отношения» после показа результата
- **THEN** текущий результат заменяется следующими отношениями

### Requirement: Отношения не повторяются в пределах цикла
Генератор SHALL выдавать все доступные отношения в случайном порядке без
повторов до исчерпания подборки. После начала нового цикла первый результат MUST
отличаться от последнего результата предыдущего цикла, если подборка содержит
более одного отношения.

#### Scenario: В подборке остаются непоказанные отношения
- **WHEN** пользователь запрашивает следующий вариант
- **THEN** генератор выбирает его только среди отношений, ещё не показанных в текущем цикле

#### Scenario: Подборка исчерпана
- **WHEN** пользователь запрашивает следующий вариант после показа всех отношений
- **THEN** генератор начинает новый перемешанный цикл без немедленного повтора последнего результата

### Requirement: Роли двух персонажей доступны игрокам
Генератор SHALL показывать на экране итоговую формулировку отношения между двумя
персонажами. Роли первого и второго игроков MUST быть доступны вспомогательным
технологиям через объявление результата; интерфейс MUST NOT добавлять персонажам
характеристики сверх выбранного отношения.

#### Scenario: Выбраны несимметричные отношения
- **WHEN** результат содержит разные роли, например врача и пациента
- **THEN** на экране показывается формулировка отношения, а объявление результата называет роль каждого из двух игроков

#### Scenario: Выбраны симметричные отношения
- **WHEN** оба персонажа находятся в одинаковой роли, например являются коллегами
- **THEN** интерфейс показывает общую связь, а объявление результата называет общую роль обоих игроков

### Requirement: Пользователь может вернуться по цепочке предыдущих результатов
После замены показанных отношений генератор SHALL позволять последовательно
возвращаться к ранее показанным результатам, шаг за шагом до самого первого.
Каждый возврат MUST восстанавливать состояние так, чтобы уже показанные отношения
не повторялись в текущем цикле. Пока не показан ни один предыдущий результат,
элемент возврата MUST быть недоступен.

#### Scenario: Пользователь случайно заменил отношения
- **WHEN** после генерации следующего варианта пользователь активирует возврат
- **THEN** интерфейс снова показывает непосредственно предшествовавшие отношения

#### Scenario: Пользователь возвращается на несколько шагов
- **WHEN** пользователь активирует возврат несколько раз подряд
- **THEN** интерфейс показывает результаты в обратном порядке их появления, пока не будет достигнут первый показанный результат

#### Scenario: Предыдущего результата ещё нет
- **WHEN** пользователь ещё не заменял показанные отношения или уже вернулся к первому результату
- **THEN** действие возврата недоступно и не изменяет состояние генератора

### Requirement: Интерфейс имеет самостоятельный и приятно анимированный образ
Страница генератора SHALL использовать формулировку отношения как центральный
образ страницы. Оформление и ключевое взаимодействие MUST быть разработаны
специально для генератора отношений и MUST NOT копировать дизайн другого
инструмента проекта. Интерфейс SHALL использовать плавные, выразительные и
согласованные анимации при генерации, возврате и взаимодействии с элементами
управления. Отклик на нажатие MAY сопровождаться коротким звуком. Движение MUST
сохранять читаемость и MUST NOT задерживать следующее действие пользователя.

#### Scenario: Отображается результат
- **WHEN** генератор показывает выбранные отношения
- **THEN** формулировка отношения читается как главная визуальная структура страницы

#### Scenario: Пользователь получает новые отношения
- **WHEN** пользователь запускает генерацию или возвращает предыдущий результат
- **THEN** результат сменяется плавной согласованной анимацией

#### Scenario: Пользователь взаимодействует с управлением
- **WHEN** пользователь нажимает, касается или активирует с клавиатуры доступную кнопку
- **THEN** интерфейс даёт заметный приятный отклик без потери читаемости и задержки следующего действия

### Requirement: Генератор доступен на поддерживаемых устройствах
Основная генерация и возврат SHALL работать с указателем, клавиатурой и касанием.
Результат MUST объявляться вспомогательным технологиям, а интерфейс MUST
оставаться пригодным для телефона и настольного экрана.

#### Scenario: Пользователь работает с клавиатуры
- **WHEN** пользователь переводит фокус на доступное действие и активирует его с клавиатуры
- **THEN** действие выполняется, а видимый фокус сохраняет понятную навигацию

#### Scenario: Пользователь получает новый результат
- **WHEN** показанные отношения изменяются
- **THEN** вспомогательная технология получает объявление нового результата

### Requirement: Генератор работает как автономная статическая страница
Генератор SHALL работать без сборки, фреймворка и сетевого запроса при прямом
открытии через `file://`, а все внутренние ресурсы и переход в каталог MUST
использовать относительные пути, совместимые с GitHub Pages.

#### Scenario: Страница открыта локально
- **WHEN** пользователь открывает `tools/relationship/index.html` через `file://`
- **THEN** данные загружаются, генерация работает и переход в каталог остаётся доступным без локального сервера

#### Scenario: Страница открыта на GitHub Pages
- **WHEN** инструмент опубликован вместе с остальным статическим сайтом
- **THEN** страница использует ресурсы и внутренние ссылки внутри текущего размещения сайта
