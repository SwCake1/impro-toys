## Requirements
### Requirement: Пользователь получает локацию одним действием
Приложение SHALL показывать одну случайную локацию после активации основной кнопки и SHALL позволять получить следующую локацию повторной активацией. Стартовый экран MUST быть готов к первой генерации без предварительной настройки.

#### Scenario: Первая генерация
- **WHEN** пользователь активирует основную кнопку
- **THEN** приложение показывает одну локацию из подборки

#### Scenario: Следующая генерация
- **WHEN** пользователь повторно активирует основную кнопку
- **THEN** приложение заменяет текущую локацию новой

### Requirement: Приложение показывает счётчик локаций
Интерфейс SHALL показывать количество локаций в подборке: текущую позицию в круге и общее число локаций (например «17 из 104»). Счётчик MUST обновляться при каждой смене локации и MUST отражать актуальный размер подборки без ручного задания числа.

#### Scenario: Отображение счётчика при генерации
- **WHEN** приложение показывает локацию
- **THEN** рядом отображается текущая позиция в круге и общее число локаций в подборке

#### Scenario: Обновление позиции внутри круга
- **WHEN** пользователь запрашивает следующую локацию
- **THEN** текущая позиция увеличивается, а по началу нового круга сбрасывается к первой при неизменном общем числе

### Requirement: Локации не повторяются внутри круга
Алгоритм генерации MUST перебирать перемешанную подборку и не повторять локацию, пока не будут показаны все остальные локации круга; по исчерпании круга подборка MUST перемешиваться заново.

#### Scenario: Перебор без повторов
- **WHEN** пользователь последовательно запрашивает локации внутри одного круга
- **THEN** ни одна локация не повторяется до того, как показаны все остальные

#### Scenario: Начало нового круга
- **WHEN** показаны все локации текущего круга и пользователь запрашивает следующую
- **THEN** приложение перемешивает подборку и начинает новый круг

### Requirement: Подборка содержит русскоязычные локации для начала сцены
Локальная подборка MUST содержать не менее 100 уникальных русскоязычных локаций, пригодных для начала импровизационной сцены, и SHALL охватывать разнообразные места — бытовые, городские, природные, фантастические и профессиональные.

#### Scenario: Проверка содержимого подборки
- **WHEN** набор локаций подготовлен к выпуску
- **THEN** в нём присутствует не менее 100 уникальных непустых локаций

#### Scenario: Согласованные примеры входят в подборку
- **WHEN** пользователь перебирает доступные локации
- **THEN** среди них присутствуют «Секретная лаборатория», «Ночной музей», «Космическая станция», «Волшебный лес», «Станция метро» и «Подводная лодка»

### Requirement: Локация отображается на перекидном табло
Интерфейс SHALL показывать выпавшую локацию в виде split-flap табло из перекидных символов, стилизованного под информационное табло вокзала или аэропорта. При смене локации символы SHALL визуально «перекручиваться» до целевого текста, создавая узнаваемую механику табло, при этом итоговый текст локации MUST оставаться полностью читаемым.

#### Scenario: Пользователь открывает приложение
- **WHEN** отображается начальный или сгенерированный экран
- **THEN** пользователь видит табло с локацией, счётчик локаций, основную кнопку и переход в каталог, все элементы читаемы

#### Scenario: Смена локации с анимацией табло
- **WHEN** пользователь запрашивает новую локацию
- **THEN** символы табло перекручиваются от прежнего текста к новому и фиксируются на читаемой локации

#### Scenario: Пользователь предпочитает сокращённую анимацию
- **WHEN** браузер сообщает о настройке `prefers-reduced-motion: reduce`
- **THEN** приложение сокращает или отключает анимацию перекручивания и показывает итоговую локацию без потери функциональности

### Requirement: Генератор доступен на поддерживаемых устройствах
Основная генерация MUST работать с указателем, касанием и клавиатурой, а интерфейс SHALL адаптироваться к телефонным и настольным экранам без горизонтальной прокрутки.

#### Scenario: Генерация с клавиатуры
- **WHEN** пользователь активирует основную кнопку стандартным клавиатурным способом
- **THEN** приложение генерирует следующую локацию и сохраняет заметный фокус

#### Scenario: Использование на телефоне
- **WHEN** приложение открыто на узком сенсорном экране
- **THEN** табло с локацией, основная кнопка и переход в каталог доступны без горизонтальной прокрутки

#### Scenario: Объявление новой локации
- **WHEN** локация меняется после действия пользователя
- **THEN** вспомогательная технология получает текстовое уведомление о новой локации

### Requirement: Приложение работает локально без сети
Страница, логика, оформление и подборка локаций MUST загружаться по относительным путям без API, внешних зависимостей и сетевых запросов. Данные локаций MUST подключаться способом, работающим при открытии через `file://` (подключаемый скрипт с массивом локаций, а не загрузка JSON через `fetch`).

#### Scenario: Запуск через file protocol
- **WHEN** пользователь открывает страницу инструмента через `file://`
- **THEN** подборка локаций загружается, генерация и переход обратно в каталог работают без подключения к интернету

### Requirement: Инструмент представлен карточкой в каталоге
Каталог инструментов SHALL содержать запускаемую карточку генератора локаций с названием и лаконичным визуальным идентификатором, ведущую на страницу инструмента по относительной ссылке.

#### Scenario: Пользователь открывает генератор из каталога
- **WHEN** пользователь активирует карточку генератора локаций в каталоге
- **THEN** браузер переходит на страницу инструмента генератора локаций
