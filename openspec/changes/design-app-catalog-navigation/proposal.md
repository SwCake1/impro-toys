## Зачем

Сейчас главная страница представляет собой простой список из одной ссылки и не формирует привлекательный, масштабируемый каталог инструментов для импровизации. Сайту нужна понятная адаптивная точка входа и единая навигация для возврата из каждого независимого инструмента, при этом интерфейс должен оставаться живым и побуждать к исследованию.

## Что изменится

- Простой список на главной странице будет заменён адаптивным каталогом карточек доступных мини-приложений.
- Каталог получит чистое оформление с лёгким театральным настроением, но без декоративного шума.
- Интерактивные анимации сделают каталог живым и оставят пространство для будущих пасхалок.
- На телефонах каталог будет содержать две колонки, а на широких экранах количество колонок будет динамически увеличиваться.
- Вся поверхность каждой карточки будет доступной ссылкой на соответствующий инструмент.
- В каждом инструменте появится единый и понятно подписанный способ вернуться в каталог.
- Внутренние ссылки останутся относительными, чтобы сайт продолжал работать локально и на GitHub Pages.

## Возможности

### Новые возможности

- `tool-catalog`: адаптивный и анимированный каталог для поиска и запуска доступных инструментов импровизации.
- `tool-navigation`: единая навигация между каталогом и отдельными страницами инструментов.

### Изменяемые возможности

Нет.

## Влияние

- Перерабатывается корневая страница `index.html`.
- В `shared/` добавляются только действительно необходимые общие стили и навигационные ресурсы.
- В `tools/bell/index.html` добавляется переход в каталог без изменения основной логики звонка.
- Формируется переиспользуемое соглашение для будущих страниц в `tools/`.
- Не добавляются фреймворки, сборка, пакетные или внешние runtime-зависимости.
