## Context

Проект состоит из независимых статических инструментов без сборки и внешних
зависимостей. Генератор первой реплики должен работать при прямом открытии через
`file://`, на GitHub Pages, на телефоне и на настольном экране. Его назначение —
дать импровизаторам сильную первую фразу, которая мгновенно задаёт ситуацию и
напряжение, оставляя развитие сцены, персонажей и сюжет полностью на их
усмотрение.

## Goals / Non-Goals

**Goals:**

- Мгновенно показывать одну яркую открывающую реплику для старта сцены.
- Предоставить не менее 200 уникальных и содержательно различных реплик.
- Присвоить каждой реплике один тон-тег из небольшого фиксированного набора и
  показывать его вместе с репликой.
- Позволить фильтровать выдачу по одному тону или брать реплики из всех тонов.
- Исключить повторы до исчерпания текущей колоды.
- Показывать прогресс — сколько реплик уже показано из общего числа в колоде.
- Создать самостоятельный визуальный образ «Софит» с приятными анимациями и
  звуком.
- Сохранить полную работоспособность через `file://` и на мобильных устройствах.

**Non-Goals:**

- Не задавать за игроков персонажей, отношения, конфликт, цель или развязку сверх
  того, что подразумевает сама реплика.
- Не добавлять множество мелких тегов, категорий, избранное, историю сессий или
  таймер.
- Не сохранять состояние между перезагрузками страницы.
- Не строить визуально перегруженный, «алиэкспрессовый» интерфейс с большим
  числом мелких деталей.

## Decisions

### 1. Реплика хранится как готовая редакторская запись с одним тоном

Каждая запись содержит текст реплики и ровно один тон-тег. Готовые записи
выбираются вместо генерации из шаблонов: так можно проверить естественность
русского языка, драматургическую силу первой фразы и корректность тона каждой
реплики.

Набор тонов фиксирован (8 категорий, каждая — одно слово): «быт», «конфликт»,
«работа», «абсурд», «романтика», «искусство», «экстрим», «эмоции». Небольшое
число крупных тонов поддерживает чистый дизайн фильтра и не превращает его в
мелкую сетку.

### 2. Данные подключаются обычным JavaScript-файлом

Список размещается рядом с инструментом в отдельном `lines.js`, который
подключается до `app.js`. Это сохраняет отделение контента от логики и не требует
`fetch`, ненадёжного для локальных файлов через `file://`.

### 3. Фильтр по тону задаёт активную колоду

По умолчанию активна колода из всех реплик. При выборе тона активной становится
колода только этого тона. Смена фильтра начинает соответствующую колоду заново с
перемешиванием. Управление тоном — крупные переключатели с состоянием «все» и по
одному на каждый тон.

### 4. Генерация использует перемешанную колоду без повторов

При старте колоды создаётся перемешанный набор индексов. Каждый новый результат
извлекается из него один раз; после исчерпания создаётся новая перемешанная
колода. Первый элемент нового цикла не совпадает с последним показанным элементом
предыдущего цикла, если в колоде больше одной реплики.

Прогресс показывает, сколько реплик уже показано из общего числа в текущей
колоде (например «Показано 12 из 200», а при фильтре — из размера выбранной
группы). Счётчик обнуляется при смене фильтра и при начале нового цикла.

### 5. Основной экран — образ «Софит»

Интерфейс строится вокруг единственного сильного акцента: крупной реплики в
центре, освещённой мягким лучом света на тёмной сцене. Деталей минимум,
пространства много. Под репликой тихо показывается её тон-тег, ниже — прогресс.
На странице одна основная кнопка «Свет софитов» и крупный фильтр тонов сверху.

Ключевое взаимодействие: при запросе новой реплики свет на миг сужается или
гаснет, затем новая реплика «выходит на свет» плавным появлением с лёгким
движением, сопровождаемым мягким звуком софита. Оттенок луча может меняться под
выбранный тон. Движение цельное и спокойное, сохраняет читаемость и не
задерживает следующее нажатие. Оформление и движение не копируют решения других
инструментов проекта (в частности, не используют перекидное табло).

### 6. Инструмент остаётся полностью автономным

Страница, стили, данные и логика находятся в `tools/first-line/`. Из общих
ресурсов используются только принятый элемент навигации и карточка каталога с
собственным графическим идентификатором. Все внутренние пути остаются
относительными.

## Risks / Trade-offs

- [Часть реплик может незаметно навязывать сюжет или персонажей] → Проверить
  каждую запись: реплика задаёт лишь исходную ситуацию и оставляет развитие
  игрокам.
- [Формально разные реплики могут быть смысловыми дублями] → Провести отдельную
  редакторскую проверку списка, а не ограничиваться сравнением строк.
- [Тон реплики можно назначить спорно] → Присваивать тон по основному
  драматическому эффекту первой фразы; допускать один тон на реплику ради
  простоты фильтра.
- [Большой список из 200 сложно проверять вручную] → Добавить статическую
  проверку количества, уникальности строк, наличия текста и корректного тона у
  каждой записи, дополнив её ручной содержательной проверкой.
- [Выразительный свет и движение могут отвлекать от реплики] → Привязать анимации
  к действию пользователя, держать реплику главным акцентом и не блокировать
  повторную генерацию на время переходов.
- [Мелкие детали или перегруженный фильтр разрушат образ «Софит»] → Ограничить
  число тонов, оставить один крупный акцент и много пустого пространства.
