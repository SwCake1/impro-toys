## ADDED Requirements

### Requirement: Генератор предоставляет большую подборку первых реплик
Генератор SHALL содержать не менее 200 содержательно различных первых реплик на
русском языке. Каждая реплика MUST задавать только исходную ситуацию для старта
сцены и MUST оставлять развитие сцены, персонажей, конфликт, цель и развязку
игрокам.

#### Scenario: Подборка проходит содержательную проверку
- **WHEN** проверяется полный набор реплик
- **THEN** он содержит не менее 200 первых реплик, каждая из которых оставляет развитие сцены игрокам

#### Scenario: В подборке отсутствуют смысловые дубли
- **WHEN** сравниваются формулировки и смысл всех реплик
- **THEN** ни одна реплика не повторяет другую только иными словами

### Requirement: Каждой реплике присвоен один тон из фиксированного набора
Генератор SHALL относить каждую реплику ровно к одному тону из небольшого
фиксированного набора тонов. Тон текущей реплики MUST показываться на экране
вместе с её текстом.

#### Scenario: Реплика показана с тоном
- **WHEN** генератор показывает выбранную реплику
- **THEN** на экране рядом с репликой отображается её тон

#### Scenario: Каждая реплика имеет допустимый тон
- **WHEN** проверяется полный набор реплик
- **THEN** каждой реплике присвоен ровно один тон из фиксированного набора

### Requirement: Пользователь получает реплику одним основным действием
Генератор SHALL показывать новую первую реплику после активации основной кнопки.
До первого запуска интерфейс MUST ясно приглашать пользователя выполнить это
действие.

#### Scenario: Пользователь запускает генератор впервые
- **WHEN** пользователь активирует основную кнопку до показа первой реплики
- **THEN** интерфейс показывает одну случайно выбранную реплику

#### Scenario: Пользователь запрашивает следующую реплику
- **WHEN** пользователь активирует основную кнопку после показа реплики
- **THEN** текущая реплика заменяется следующей

### Requirement: Пользователь может фильтровать реплики по тону
Генератор SHALL позволять пользователю выбрать один тон или режим «все тоны».
Пока выбран конкретный тон, генератор MUST выдавать только реплики этого тона.
Смена выбранного тона MUST начинать новую колоду соответствующей группы.

#### Scenario: Выбран конкретный тон
- **WHEN** пользователь выбирает один тон и запрашивает реплику
- **THEN** генератор показывает реплику только выбранного тона

#### Scenario: Выбран режим всех тонов
- **WHEN** пользователь выбирает режим «все тоны» и запрашивает реплику
- **THEN** генератор показывает реплику из всей подборки без ограничения по тону

#### Scenario: Пользователь меняет фильтр
- **WHEN** пользователь переключает выбранный тон
- **THEN** генератор начинает новую перемешанную колоду выбранной группы

### Requirement: Реплики не повторяются в пределах цикла
Генератор SHALL выдавать все реплики активной колоды в случайном порядке без
повторов до её исчерпания. После начала нового цикла первый результат MUST
отличаться от последнего результата предыдущего цикла, если колода содержит более
одной реплики.

#### Scenario: В колоде остаются непоказанные реплики
- **WHEN** пользователь запрашивает следующую реплику
- **THEN** генератор выбирает её только среди реплик активной колоды, ещё не показанных в текущем цикле

#### Scenario: Колода исчерпана
- **WHEN** пользователь запрашивает следующую реплику после показа всех реплик активной колоды
- **THEN** генератор начинает новый перемешанный цикл без немедленного повтора последнего результата

### Requirement: Интерфейс показывает прогресс по текущей колоде
Генератор SHALL показывать, сколько реплик уже показано из общего числа реплик в
текущей активной колоде. Показатель прогресса MUST обнуляться при смене фильтра и
при начале нового цикла.

#### Scenario: Пользователь получает реплики подряд
- **WHEN** пользователь последовательно запрашивает реплики в пределах одной колоды
- **THEN** интерфейс показывает растущее число показанных реплик и общее число реплик в текущей колоде

#### Scenario: Пользователь меняет фильтр
- **WHEN** пользователь переключает выбранный тон
- **THEN** прогресс начинается заново и отражает размер выбранной группы

### Requirement: Интерфейс имеет самостоятельный образ «Софит» с анимацией и звуком
Страница генератора SHALL использовать текст реплики как центральный образ
страницы в виде освещённой сцены с одним мягким лучом света и минимумом деталей.
Оформление и ключевое взаимодействие MUST быть разработаны специально для этого
инструмента и MUST NOT копировать дизайн другого инструмента проекта. Интерфейс
SHALL использовать плавную анимацию появления новой реплики и SHALL воспроизводить
мягкий звук при её получении. Движение MUST сохранять читаемость и MUST NOT
задерживать следующее действие пользователя.

#### Scenario: Отображается реплика
- **WHEN** генератор показывает выбранную реплику
- **THEN** текст реплики читается как главный визуальный акцент освещённой сцены

#### Scenario: Пользователь получает новую реплику
- **WHEN** пользователь запускает генерацию новой реплики
- **THEN** новая реплика появляется плавной анимацией и сопровождается мягким звуком

#### Scenario: Пользователь взаимодействует с управлением
- **WHEN** пользователь нажимает, касается или активирует с клавиатуры доступную кнопку
- **THEN** интерфейс даёт заметный отклик без потери читаемости и задержки следующего действия

### Requirement: Генератор доступен на поддерживаемых устройствах
Основная генерация и фильтр по тону SHALL работать с указателем, клавиатурой и
касанием. Результат MUST объявляться вспомогательным технологиям, а интерфейс
MUST оставаться пригодным для телефона и настольного экрана. Все тап-цели MUST
использовать `touch-action: manipulation`, чтобы быстрый двойной тап не вызывал
масштабирование страницы.

#### Scenario: Пользователь работает с клавиатуры
- **WHEN** пользователь переводит фокус на доступное действие и активирует его с клавиатуры
- **THEN** действие выполняется, а видимый фокус сохраняет понятную навигацию

#### Scenario: Пользователь получает новую реплику
- **WHEN** показанная реплика изменяется
- **THEN** вспомогательная технология получает объявление новой реплики

### Requirement: Генератор работает как автономная статическая страница
Генератор SHALL работать без сборки, фреймворка и сетевого запроса при прямом
открытии через `file://`, а все внутренние ресурсы и переход в каталог MUST
использовать относительные пути, совместимые с GitHub Pages.

#### Scenario: Страница открыта локально
- **WHEN** пользователь открывает `tools/first-line/index.html` через `file://`
- **THEN** данные загружаются, генерация и фильтр работают, а переход в каталог остаётся доступным без локального сервера

#### Scenario: Страница открыта на GitHub Pages
- **WHEN** инструмент опубликован вместе с остальным статическим сайтом
- **THEN** страница использует ресурсы и внутренние ссылки внутри текущего размещения сайта
