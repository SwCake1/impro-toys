## ADDED Requirements

### Requirement: Каждый инструмент позволяет явно вернуться в каталог
Каждая отдельная страница инструмента SHALL содержать единообразно расположенный навигационный элемент, подписанный названием места назначения, а не представленный одной неподписанной иконкой.

#### Scenario: Пользователь возвращается из инструмента
- **WHEN** пользователь активирует элемент перехода в каталог на странице инструмента
- **THEN** браузер напрямую переходит на корневую страницу каталога

#### Scenario: Инструмент открыт по прямой ссылке
- **WHEN** пользователь открывает URL инструмента, не посещая перед этим каталог
- **THEN** элемент перехода всё равно ведёт в каталог и не зависит от истории браузера

### Requirement: Навигация не мешает работе инструмента
Общий элемент перехода в каталог SHALL оставаться заметным и доступным на настольных и мобильных экранах, сохраняя самостоятельный визуальный стиль и основную интерактивную область каждого инструмента.

#### Scenario: Навигация отображается поверх полноэкранного инструмента
- **WHEN** инструмент использует собственный полноэкранный интерфейс
- **THEN** элемент перехода в каталог остаётся видимым и доступным, не перекрывая основной элемент управления или настройки инструмента

### Requirement: Навигация переиспользуется без системы сборки
Соглашение о навигации SHALL позволять будущим отдельным страницам инструментов использовать статический HTML и общие ресурсы без фреймворка, пакетных зависимостей или этапа сборки.

#### Scenario: Будущий инструмент добавляет переход в каталог
- **WHEN** в проект добавляется новая статическая страница инструмента
- **THEN** она может реализовать ту же навигацию средствами обычного HTML и общих ресурсов репозитория
