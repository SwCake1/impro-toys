## Context

Проект — статический сайт без сборки и пакетного менеджера (см. `AGENTS.md`). Каждый инструмент живёт в своей папке `tools/<name>/` с локальными логикой и данными; общие стили каталога — в `shared/`. Сайт открывают как через `file://` (двойной клик по `index.html`), так и на GitHub Pages, поэтому все пути относительные, а данные подключаются без сетевых запросов.

Список из 104 локаций уже лежит в `tools/locations/locations.txt`. Существует близкий по задаче инструмент `tools/emotion/` («одна кнопка → случайный элемент без повторов внутри круга») и свежий `tools/rap-battle/`, в котором данные вынесены в отдельный файл `topics.js` (`window.RAP_BATTLE_TOPICS = [...]`) и подключаются тегом `<script>`. Пользователь хочет для генератора локаций собственный визуальный характер — split-flap табло, — а не копию существующих инструментов.

## Goals / Non-Goals

**Goals:**
- Одна крупная кнопка, выдающая случайную локацию без повторов внутри круга.
- Уникальное оформление split-flap табло с анимацией перекручивания символов.
- Данные локаций в отдельном файле, работающем через `file://`.
- Поддержка указателя, касания, клавиатуры; адаптивность под телефон и десктоп; учёт `prefers-reduced-motion`.
- Единообразная интеграция в каталог: новая карточка + переход «Все инструменты».

**Non-Goals:**
- Категории/группы локаций и окраска фона по типу (в этой итерации не делаем).
- Модификаторы сцены (вторая строка-твист) — отклонено пользователем.
- Смешивание с персонажами или другими генераторами.
- Серверная часть, сборка, внешние библиотеки.

## Decisions

### Хранение данных: `locations.js` с `window.LOCATIONS`
Данные выносятся из `locations.txt` в `tools/locations/locations.js` как `window.LOCATIONS = ["Секретная лаборатория", ...]`, подключаются через `<script src="./locations.js">`.

- **Почему так:** приём уже применён в `rap-battle/topics.js` и гарантированно работает через `file://`.
- **Альтернатива — `fetch('locations.json')`:** отклонена, потому что браузеры блокируют `fetch` локальных файлов через `file://` (CORS), и сайт открывают именно двойным кликом. Дало бы пустой экран без локального сервера.
- **Альтернатива — данные прямо в `app.js`:** отклонена, пользователь хочет данные отдельным файлом; вынесение улучшает читаемость и упрощает пополнение списка.
- `locations.txt` остаётся в репозитории как исходный человекочитаемый источник; `locations.js` генерируется из него.

### Алгоритм выдачи: перемешанный перебор без повторов
Формируем перемешанную копию `LOCATIONS`, идём по индексу; дошли до конца — перемешиваем заново и начинаем новый круг. Это исключает повторы внутри круга и подряд идущие дубли.

- **Почему так:** тот же подход, что в `emotion/app.js` (проверенный в проекте), лучше чистого рандома, который может выдать одну локацию несколько раз подряд.

### Визуальный компонент: split-flap табло на CSS/JS
Табло — ряд(ы) фиксированного числа «барабанов»-ячеек. При смене локации каждая ячейка анимированно «долистывает» символы до целевого; колонки стартуют с нарастающей задержкой, создавая волну перекручивания слева направо. Число ячеек рассчитано на самую длинную локацию (например, «Станция для путешественников во времени»); при необходимости — перенос на несколько строк.

- **Алфавит перекрутки:** кириллица (33 буквы), пробел и дефис. Чтобы анимация не была слишком долгой, ячейка не обязана прокручивать весь алфавит — достаточно короткого листания к цели с ограничением длительности.
- **Читаемость:** финальное состояние — статичный, полностью читаемый текст; анимация только декоративна.
- **Доступность:** при `prefers-reduced-motion: reduce` анимация перекручивания сокращается/отключается, локация показывается сразу. Смена локации объявляется вспомогательным технологиям (`aria-live`).
- **Почему табло:** тематически совпадает с идеей «места назначения», даёт живую механику именно на нажатие, реализуется без картинок и зависимостей, визуально не похоже на другие инструменты каталога.
- **Альтернативы (колода карточек, неоновая вывеска, указатель/пин):** рассмотрены и отклонены пользователем в пользу табло.

### Структура файлов и стили
`tools/locations/`: `index.html`, `app.js`, `locations.js`, и при необходимости `styles.css` (как в `rap-battle`). Табло имеет заметную собственную стилистику, поэтому специфические стройки логично держать локально, а не в `shared/`.

- Карточка в корневом `index.html` со своей иконкой в `shared/assets/` (например `locations.svg`) и собственным акцентным цветом через `--card-accent`.
- Навигация «Все инструменты» и относительные пути — как в остальных инструментах.

## Risks / Trade-offs

- **Длинные локации на узком экране** → табло переносит текст на несколько строк и/или адаптирует размер ячеек; проверить на мобильном самую длинную локацию.
- **Долгая анимация из-за большого кириллического алфавита** → ограничить длительность/шаг листания, не прокручивать весь алфавит целиком.
- **Расхождение `locations.txt` и `locations.js`** → фиксируем `locations.js` как источник для рантайма; `.txt` считается человекочитаемым исходником, обновляется вместе с `.js` (или удаляется во избежание дублирования — решается при реализации).
- **Перегрузка анимацией для части пользователей** → соблюдаем `prefers-reduced-motion` и не блокируем генерацию, если что-то в анимации не сработало.

## Open Questions

- Оставлять ли `locations.txt` в репозитории после конвертации, или удалить, чтобы не было двух источников? (Уточнить при реализации; по умолчанию — оставить как исходник.)
- Нужен ли звук перекручивания табло, или инструмент работает молча? (По умолчанию — без звука; можно добавить опционально, как в других инструментах.)
